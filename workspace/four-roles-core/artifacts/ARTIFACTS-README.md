# 四角色协作 — 产出文档与变更记录

本目录存放各角色的**持久化产出**与**文档变更记录**，便于测试与审计。

| 文档 | 负责角色 | 说明 |
|------|----------|------|
| 产品需求变更记录.md | 产品经理 | 每次需求产出或验收结论变更时追加 |
| 架构变更记录.md | 技术架构师 | 每次方案或关键节点把关结论变更时追加 |
| 测试记录.md | 测试总监 | 可测性、每轮测试执行产出时追加 |
| 测试报告.md | 测试总监 | 按 run/版本汇总的测试范围、结果与发布建议 |
| 测试文档变更记录.md | 测试总监 | 测试策略/用例/报告等文档变更时追加 |

## 使用要求

- 每完成对应阶段且产出有变更时，**必须**在对应文档的「变更记录表」中追加一行（时间、runId、阶段、变更摘要、关联产出）。
- 运行日志（runs/<runId>.json）中可引用本目录下的文档路径作为 outputFullRef。
